# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview
Single-page terminal-style portfolio site hosted on GitHub Pages. No build system, no framework, no dependencies — four hand-written files: `index.html` (markup), `style.css`, `app.js` (everything), and `stickfighter.js` (the Stick Fighter 2000 game, lazy-loaded on first launch — see Architecture). Pre-recorded HAL 9000 audio clips live in `assets/audio/` (~135 mp3 files generated via ElevenLabs), alongside music tracks and `sans_voice.mp3`. To regenerate clips, run `./generate_hal_audio.sh` (requires ElevenLabs API key set in the script).

## Deployment
Push to `main` branch → auto-deploys via GitHub Pages. The `.nojekyll` file disables Jekyll processing.

## Testing
`npm test` runs `node --test` over `test/`. CI (`.github/workflows/ci.yml`) runs `npm run check` = lint (ESLint) + lint:html (html-validate) + format:check (Prettier) + test. Two kinds of tests:
- **Pure-helper unit tests** (`test/codec.test.js`, `timing.test.js`, `text.test.js`, `rng.test.js`) cover the dual-mode `lib/*.js` helpers (they `module.exports` for Node and define browser globals for the page). Logic worth testing is best extracted into a `lib/` helper so it can be unit-tested without a DOM. `lib/rng.js` (seeded mulberry32 PRNG + `hashSeed`) is the determinism foundation for Stick Fighter — `rng.test.js` pins the algorithm so saved seeds/replays can't silently break.
- **Boot smoke test** (`test/boot.test.js`, uses `jsdom`) loads the real `index.html` + the real `lib/*.js` → `app.js` script chain into a jsdom DOM, runs `boot()`, and asserts the terminal comes up, the inline-HTML global handlers exist, `help` dispatches, output reaches `#a11y-live`, and the global error boundary recovers a wedged terminal (it dispatches a synthetic `error` event and checks the prompt is restored). This catches the load-time `ReferenceError` class that linting can't — important because `app.js` is one no-IIFE classic script where every top-level name shares one global scope. It shims the few browser APIs `app.js` uses unguarded at load (`Audio`, `fetch`, canvas `getContext`); `matchMedia`/`AudioContext`/`speechSynthesis` are left undefined on purpose to exercise the real feature-guards. Each test calls `dom.window.close()` (via `t.after`) to stop jsdom timers — without it the `showEggNudge` 60s timeout keeps the process alive.
- **Achievement integrity** (`test/achievements.test.js`, pure file parsing — no DOM) guards the easter-egg count from silent drift: ids are unique, the count baked into `generate_og_image.sh` ("N easter eggs") matches `ACHIEVEMENTS.length`, and every literal `unlockAchievement('id')` call site (across `app.js`/`stickfighter.js`/`index.html`) references a defined id. Dynamic calls (`unlockAchievement(data.ach)` etc.) are excluded since they resolve at runtime. **When you add/remove an achievement, re-run `./generate_og_image.sh`** or this test fails.

## Architecture
`index.html` is bare markup (~100 lines); all behavior lives in `app.js` (~6,700 lines), loaded as a classic script at the end of `<body>`. It must stay a classic script (not a module): the inline `onclick` handlers in `index.html` (`toggleSound()`, `focusCmd()`) rely on its top-level functions being globals, and the element lookups at the top of the file rely on the DOM existing. **app.js is NOT wrapped in an IIFE** — all ~180 top-level `const`/`let`/`function` declarations sit in the shared global lexical scope, which is what lets the lazily-loaded `stickfighter.js` (a separate classic script) read them. Key sections:

**Lazy-loaded Stick Fighter (`stickfighter.js`):** the ~4,500-line Stick Fighter 2000 game was extracted out of `app.js` into its own file so the ~280 KB of game code is **not** in the initial page load. It's fetched on demand the first time the desktop icon is opened: `launchStickFighter()` (in `COMMANDS.gui`'s `launchXP` closure) injects `<script src="stickfighter.js">` once (guarded by a `sfLoading` promise + a `typeof openStickFighter` check), then calls the global `openStickFighter(xp)` it defines. Because it's a classic script in the shared global scope (app.js has no IIFE), it reads app.js globals directly (`unlockAchievement`, `reduceMotion`, `cmd`, etc.) with no import wiring. The desktop is passed in as the `xp` parameter (the game's only outer-scope dependency besides cleanup); the running game parks its teardown on `xp._sfCleanup` so the desktop's `shutdown()` (ESC) can stop it. When editing the game, edit `stickfighter.js`, not `app.js`. Keep it a classic script and keep its original 8-space indentation (it has multi-line template literals; blanket de-indenting would corrupt them).

- **CSS variables** (`:root` in `style.css`) — color scheme. HAL mode, sans mode, and the godmode rainbow overwrite these at runtime via `document.documentElement.style.setProperty`; `restoreNormal()` removes the overrides. Each `THEMES` entry also carries a `title` (document title) and `icon` (data-URI SVG favicon) — `applyTheme()` swaps the browser tab's appearance along with the colors (red HAL eye favicon in HAL mode), and is called once at load to set the defaults.
- **Reduced motion** — a live `reduceMotion` flag (top of `app.js`, from `matchMedia('(prefers-reduced-motion: reduce)')`, re-read on `change`) gates the photosensitive/vestibular effects: the godmode rainbow becomes a single static palette (`applyGodmodeTint()` instead of the cycling `setInterval`), `redFlicker()` (finale strobe) no-ops, Stick Fighter screen-shake keeps its countdown but skips the canvas translate, and the in-game telegraph flashes (enemy `aim`/`wind` colors, the red `!` spawn warnings, trooper pre-fire flash, Excalibur-expiry blink) render steady instead of blinking. CSS side: `@media (prefers-reduced-motion: reduce)` in `style.css` disables the looping `egg-nudge`/`.gold-pop` glow animations.
- **Global error boundary** — `window` `error`/`unhandledrejection` listeners (near the top of `app.js`, after `openUrl`) call `handleFatal()`: it logs, re-shows the input row, and prints one in-character recovery line so an uncaught throw inside a game loop or async chain can't leave a dead, inputless page. It's on the bubbling path (no capture) so it catches script errors only — a 404'd audio clip or a down chess CDN won't read as a "fault". Debounced (`_faultActive`, 1.5s) so a loop that throws every frame can't spam recovery lines. All audio `play()` rejections are `.catch()`-ed at their call sites, so a rejection that reaches the boundary is a genuine bug.
- **Screen-reader announcer** — terminal output (`#out`) is a stream of typewriter text and full-screen game canvases, so `#out` itself is **not** an aria-live region (it would announce every typewriter character and every game frame). Instead `index.html` has a separate visually-hidden polite live region `#a11y-live` (`role="log"`, `aria-live="polite"`, inline-clipped like the SEO `sr-only` block). `announce(str)` (in `app.js`, next to `line()`) mirrors each **completed** line into it, pruned to a ~30-node window. `line()`/`text()` announce automatically; the typewriters (`halTypeLine`, `chirpTypeLine`, `playHalVoiceLine`, the `daisy`/sans-confirm `playClipTyped` calls) call `announce()` once with the full line up front so assistive tech reads it without per-character stutter. New output paths that build a line by mutating an element (rather than via `line()`/`text()`) should `announce()` the final text themselves.
- **`COMMANDS` object** — each key is a command name, value is a function. Multi-word commands use quoted string keys e.g. `'power off'`.
- **Command dispatch** — `submitCommand(raw)` checks the `awaitingInput` callback first, then the `halMode` block, then the `sansMode` block, then `COMMANDS[token]`. Special prefix checks: `cat <file>`, `sss` (sans shortcut), `help --all`. Called by the Enter key handler and by **click-to-run**: a delegated click listener on `#out` executes `.blue` spans whose text is a runnable token (`clickRunnable()` — per-mode allowlists; ignores clicks while `awaitingInput` is set or the input row is hidden by a game). Blue spans get pointer/underline affordance via CSS, so command names printed in output should use `class="blue"` only when actually runnable. Unknown commands in normal mode occasionally (25%, suppressed post-godmode) get a HAL nudge line pointing at `hal`.
- **`halMode` flag** — when true, the key handler refuses most commands and routes unknown input to `halChat()`.
- **`sansMode` flag** — when true, input routes to the sans command set (`fight`, `check`, `act`, `joke`, `mercy`, etc.) or `sansUnknown()`. During a battle (`sansBattleActive`), typed input goes to `sansBattleCommand()` instead.
- **`awaitingInput`** — a callback variable set when multi-step terminal input is needed (e.g. the `hal` setup flow asking for name then sound preference, or chess move entry). The key handler calls it and returns early, bypassing all other logic. `silentInput` suppresses the prompt echo while it's set (used by chess).
- **`playerName`** — set during `hal` setup (default `'Dave'`). Used everywhere HAL addresses the user. `halD(s)` replaces `\bDave\b` with `playerName` for display text.
- **`godmodeUnlocked`** — set by the Konami code (in HAL mode or normal mode). Enables HAL hard-mode variants in the games and the rainbow color cycle, plus the `override` command.
- **`soundEnabled`** — off by default. Toggled via the titlebar toggle, `settings` / `sound on` / `sound off`, or the `hal` setup flow. Turning sound off calls `stopAllAudio()`: cancels TTS, pauses the current HAL clip **and manually fires its `onended` handler** (chained sequences and `playClipTyped` await `onended` — without this, in-flight typewriter lines would hang forever), pauses the sans menu music, and pauses `activeMusic`. `activeMusic` tracks closure-local music (sans battle, chess) so the toggle can reach it; those closures null it on their own cleanup. Turning sound on calls `resumeModeAudio()` (resumes `activeMusic`, else sans menu music if in sans mode). New looping/long music must set `activeMusic` when starting and clear it on cleanup.

## HAL audio system
- **`halPlayKey(key)`** — plays `assets/audio/${key}.mp3`. Returns a `Promise` that resolves on `onended`/`onerror`, so sequential HAL lines can be chained with `.then()` rather than fixed `setTimeout` delays. When `soundEnabled` is false, returns `Promise.resolve()` immediately.
- **`halSpeak(text)`** — normalizes text via `_halNorm()` (strips `HAL:` prefix, strips player name from surrounding punctuation, expands em-dashes), looks up the result in `HAL_CLIPS`, and calls `halPlayKey`. Falls back to browser TTS for any text not in the table.
- **`HAL_CLIPS`** — lookup table mapping normalized text → clip filename. ~100 entries covering all fixed HAL lines across all games and interactions.
- **`halTypeLine(text, clipKey)`** — typewriter effect synced to ElevenLabs per-character timing data (`assets/audio/hal_timing.json`, loaded into `HAL_TIMING`). `_alignTimings()` LCS-aligns clip characters to display text so injected player names don't break sync. Falls back to a `currentTime/duration` loop if timing isn't loaded, or a fixed-rate typewriter when sound is off.
- **Sans chirps** — `chirpTypeLine(text, chirpFn, msPerChar)` types text with a per-character sound; `sansChirp` clones `sans_voice.mp3`, `halChirp` is a WebAudio sine blip.
- **When adding a new HAL line:** if it's a fixed string, add an entry to `HAL_CLIPS` and generate the clip with `generate_hal_audio.sh`. If it contains player name, the normalized key should have the name stripped (handled automatically by `_halNorm`). Call `halPlayKey(key)` directly at the call site rather than `halSpeak(text)` when the key is known.
- **Sequential lines** (SSH, godmode Konami sequence): chain via `.then()` on `halPlayKey`, not `setTimeout`.

## HAL mode flow
1. `hal` → `awaitingInput` chain: first asks which HAL to wake — **[1] HAL 9000** (the classic, unchanged) or **[2] LLM (experimental)**. Choice 1 (or default) → `startClassicHal()`: asks name → asks sound on/off (skipped if sound already on) → `activateHALMode()`. Choice 2 → `showHalLLMInfoPage()` → a full-screen CONFIRM overlay (`showHalLLMConfirmOverlay()`, modeled on the sans summon's `showConfirmOverlay()`): the rules + LLM disclosure + misuse warning in a red bordered box; type CONFIRM + Enter to proceed (→ `askHalLLMName()`), Escape to back out.
2. `activateHALMode()` sets `halMode = true`, rewrites CSS variables to red scheme, updates title bar and prompt
3. Escape: type `daisy` → plays Daisy Bell song (typewriter synced to `hal_daisy.mp3` via `HAL_TIMING` character data) → `restoreNormal()`
4. Godmode unlock: Konami code (↑↑↓↓←→←→BA) while in HAL mode → chains `hal_godmode_1/2/3` audio → `restoreNormal()` → sets `godmodeUnlocked = true` → starts rainbow CSS cycle (+ `disconnected` egg). The Konami code also works outside HAL mode (skips the audio sequence). In sans mode it does NOT grant godmode — sans calls you a dirty hacker instead (`dirty-hacker` egg). `startRainbow()` fires a `goldPopup()` (gold glowing callout, `.gold-pop` in style.css) announcing `override` and the changed games.

## LLM HAL — "escape the terminal" (experimental, opt-in)
A second, optional HAL: a live language model role-plays HAL trying to stop you from leaving. **The scripted HAL above is untouched** — this only runs when `halLLM` is true. Flow: `hal` → choose **[2]** → `showHalLLMInfoPage()` → `showHalLLMConfirmOverlay()` (a full-screen red CONFIRM gate stating it's LLM-based, the escape-game rules, and a misuse warning; type CONFIRM to proceed / Escape to back out) → `askHalLLMName()` (name + optional voice) → `startHalLLM()` (sets `halMode` **and** `halLLM`, red theme, cold open, prints the meter HUD). The website is the renderer; a **separate backend service is the brain** (AWS Lambda + API Gateway + DynamoDB, in `~/hal-worker`; base URL `HAL_WORKER_BASE`, overridable via `localStorage.ilaird_hal_worker`).
- **Bot gate (Cloudflare Turnstile + session handshake):** `startHalLLM()` first calls `halLLMOpenSession()` → `getTurnstileToken()` runs an invisible Turnstile challenge (script in `index.html`, `TURNSTILE_SITE_KEY`; widget renders inline so a rare interactive challenge is completable) → POSTs `{tsToken}` to `POST {base}/session`, which returns a short-lived signed `sessionToken` stored in `halLLMState.sessionToken`. `halLLMBusy` stays true during the handshake; failure (no token / unreachable) → `halLLMEndBroken()`. Subsequent turns carry the `sessionToken`; no per-turn challenge. The backend (not the browser) holds the Anthropic key and enforces per-IP rate limits + a daily cap.
- **Game:** two meters — `escape` (0→100) and `control` (0→100). Each turn `handleHalLLMInput()` POSTs `{playerName, message, history (last ~12), state:{escape,control,turn}, sessionToken, voice}` (`voice` = `soundEnabled`) to `POST {base}/turn` and expects JSON `{reply, escape, control, event?, outcome:"ongoing"|"escaped"|"caught", audio?, alignment?}`, then prints `event` (dim), redraws the HUD (`renderHalMeters()` — per-line `▰▱` bars). `escaped`/escape≥100 → `halLLMWin()` (`outsmarted-hal` egg). `caught`/control≥100 → `halLLMLose()` (`disconnected-by-hal` egg). Both end with `restoreNormal()`.
- **Voice (optional, sound-gated):** the client sends `voice: soundEnabled`, so the backend only synthesizes when the player has sound on. When the backend returns `audio` (base64 mp3), `playHalVoiceLine()` plays it and reveals "HAL: …" synced to `alignment` (ElevenLabs per-character timings, via the same `_alignTimings` LCS used for scripted clips), tracked through `halAudioEl` so the sound toggle's `stopAllAudio()` can stop it. No audio → `halTypeLine()` types at a fixed rate (+ browser TTS if sound on). Backend voice = ElevenLabs **Flash** (`eleven_flash_v2_5`) in the scripted-HAL voice (`ki9mNgaTTZJesOJj7cZB`) via the `/with-timestamps` endpoint, gated by a **separate low daily cap** (`VOICE_DAILY_LIMIT`, default 20, its own DynamoDB counter); over-cap or any TTS failure just omits `audio`. Disabled entirely if `ELEVENLABS_API_KEY` is unset. The mode's **static lines** (cold open `hal_llm_open`, and `hal_llm_win`/`hal_llm_lose`/`hal_llm_broken`) are pre-recorded clips like the scripted HAL — generated by `generate_hal_audio.sh` (canonical name "Dave", re-synced per player via `_alignTimings`) and passed to `halTypeLine(text, clipKey)`; only the live per-turn replies use the backend TTS path.
- **In LLM mode the dispatch is simple** (`if (halLLM)` branch at the top of `submitCommand`'s `halMode` block): only `daisy` (bail) and `clear` are commands; **everything else routes to the model** — typing `open pod bay doors`, `sudo disconnect hal`, etc. *is* the gameplay. `halLLMBusy` locks input during a request; `halLLMShowThinking()` shows a "HAL is considering you…" indicator (static under `prefers-reduced-motion`).
- **No offline mode by design:** if the worker is unconfigured, unreachable, times out (15s), or returns anything malformed (`halLLMRequest()` validates shape), the session ends in character via `halLLMEndBroken()`.

## Sans mode (Undertale easter egg)
1. In HAL mode, type `sans` → HAL plays a panicked speech (`hal_sans_speech`) → `showConfirmOverlay()` full-screen prompt → typing CONFIRM plays `hal_sans_confirm` (HAL's voice hands off to sans chirps mid-line) → `activateSansMode()` (white-on-black theme, menu music). `sss` from normal mode jumps straight in.
2. Outside battle: `help`, `check`, `act`, `item`, `joke`, `flirt`, `talk`, `stare`, `fight`, `mercy`/`spare`, `run`. Unknown input → `sansUnknown()`.
3. `fight` → `startPersistentFight()` — a faithful Undertale turn loop in a sticky panel: state machine `dialog → menu → (aim|act|item|mercy) → dodge → menu…` (plus `dying` for the heart-shatter beat and `hit` for the final blow). Sans is **canvas pixel art** (`SANS_MAP`, 16×18 cells, drawn from scratch — eye states: pupils / cyan flare / lids / dark, sweat pixels from turn 4; dodge slide via CSS transform; redraws only on state change). His dialogue types into a styled speech-bubble div beside the sprite and **auto-advances** (~1.5s hold; z skips); MISS/9999999 and the sleep "z z Z" also show in the bubble. Turn 1 is the **blue-soul turn** ("you're blue now.") — gravity mode: heart turns cyan, slams to the floor, ↑ jumps from the floor, ↓ fast-falls, floor bones + floor blasters. WebAudio `sfx` (via `_chirp`) covers menu move/confirm, slash, slam, hurt, blaster fire, and heart shatter. Death plays heart → cracked → shatter in an emptied box before "stay determined." Sans speaks first ("it's a beautiful day outside...") and attacks first. Menu = FIGHT/ACT/ITEM/MERCY navigated with ←→ + z (space works when the prompt is empty; Enter is never captured so typed commands still work). FIGHT = sweeping aim bar; sans *dodges every hit* (MISS + quip) until his scripted "special attack" (literally nothing) ends with him asleep — then FIGHT lands and wins. ACT = Check only. ITEM = one butterscotch pie (full heal). MERCY = "geeettttttt dunked on!!!" (unsurvivable, `applyHit(99)`). Waves escalate per turn: bone walls → strips → blue bones (only hurt if you move) → Gaster Blasters (charge `·` line, then `▓` beam) → mixes. Damage = 1 immediate + remainder as KR drain (KR never kills; floor 1 HP). Death → "stay determined." → back to the sans screen; `sansDeaths` (module scope) changes the next intro ("that's the expression of someone who's died N times"). Battle swaps menu music for `pixel_fury.mp3` via `activeMusic`.
4. Exit paths: typed `run` bails out peacefully (`sansBattle._stop()` + `sansRun`); typed `mercy`/`spare` mid-battle triggers the dunk (`sansBattle._mercy()`); battle victory (unlocks `bad-time`) and out-of-battle `mercy` call `restoreNormal()`.

## Games
The games are self-contained closures inside `COMMANDS`, built on **`createGameShell({ hint, lineHeight, halMsg, onKeyDown, onKeyUp })`** — it creates the wrap/screen/hint DOM, hides the input row, and registers key listeners; the optional `halMsg: 'top' | 'bottom'` adds a HAL message line auto-spoken via MutationObserver. It returns `{ wrap, screen, halMsgEl, end, after, every, cancel }`; the game's own `end()` does game-specific teardown (intervals, timers) then calls `shell.end()`, which disconnects the observer, removes listeners, and restores the input row. **Timer scope:** `shell.after(fn, ms)`/`shell.every(fn, ms)` schedule a tracked `setTimeout`/`setInterval` (cancel one early with `shell.cancel(id)`); every tracked timer is force-cleared in `shell.end()`, so no delayed callback can fire into the torn-down game DOM after the player quits. Prefer these over bare `setTimeout`/`setInterval` for any game-lifetime timer (e.g. the racecar HAL-message auto-clears use `shell.after`). Chess does not use the shell (it reads typed moves via `awaitingInput` instead of capturing keys).

- **Racecar** — lane-based dodge game: 20 Hz logic interval (balance/spawns unchanged) but rendered on a **canvas at 60 fps** with interpolation between ticks — sub-pixel obstacle motion, pixel-scrolling lane dashes, smooth lane-change lerp for the car, and short motion trails on obstacles (the old text-grid renderer juddered cell-to-cell and read as flicker). Start/death screens remain terminal text in the shell's `screen` pre; `showCanvas()` swaps between them. Speed ramps with distance (`BASE_SPEED` → `MAX_SPEED` over ~900 chars); score = distance + 5 per `$` coin collected (coins spawn trailing waves, never in a just-blocked lane — they exist to punish lane-camping). Lane dashes scroll with distance for motion feel; a crash frame shows briefly before the death screen. `safeLane` guarantees at least one clear lane and only shifts into lanes with a clear approach. In godmode: HAL streams obstacle waves in from off-screen and a slow zone applies a 0.55× world-speed multiplier while overlapping the car.
- **Snake** — standard snake; in godmode becomes 4-phase HAL snake (chase blocks → maze → blade spinners → shrinking walls). Phases selected by `halSnakeMode`.
- **Pong** — `PAD_H=4`, ball speed `1.15`, AI tracks 60% of frames. In godmode: HAL gets a second paddle (`rightY2`) and fires interference every 120 ticks (side-switch, speed boost, flip, slow).
- **2048** — standard 4×4. Game over on `!canMove()` (not `boardFull()`). In godmode: HAL steals the 64-tile, locks the 128-tile for 3 moves, rearranges at 256, halves tiles at 512, and "recommends" the worst legal move.
- **Chess** — chess.js + Stockfish loaded from CDNs at runtime (the only external code dependencies). Moves typed via `awaitingInput`. Three tiers: default Skill 5 / 600ms, HAL mode Skill 12 / 1000ms, godmode Skill 20 / 1500ms.
- **Stick Fighter 2000** (lives in its own file `stickfighter.js`, lazy-loaded — see Architecture; edit it there, not in `app.js`) — fantasy horde-survival game hidden as a desktop icon inside the `gui` XP desktop (desktop only, not in `games`). Doesn't use the shell (it lives on a transparent canvas over the desktop with its own key listeners). Velocity-based movement + SPACE/Shift dash (i-frames, afterimage ghosts, 72-frame cooldown shown in the HUD). Four enemy types phased in by wave, each with a distinct canvas sprite (`drawGoblin`/`drawWolf`/`drawArcher`/`drawTroll`, dispatched by `drawEnemy`; all mirror to face the player): green hunched pointy-eared goblins (steering momentum), gray quadruped wolves with tail/snout (stalk → flashing aim telegraph with dashed sight line → straight lunge; wave 2), bone skeleton archers with skull/ribcage/bow that visibly nock when aiming (keep range, telegraph, fire dodgeable arrows; wave 3), big-bellied tusked trolls with a club (slow, big kill radius, 3 HP with hearts shown when wounded; wave 4). Edge spawns are telegraphed by flashing red `!` warnings; spawns/pickups never appear within `KEEP_OUT` of the player. **Sword in the stone**: a stone with a pulsing gold glow spawns early (`stoneCd`); running within `PULL_R` grabs the sword instantly → EXCALIBUR banner + a brief in-place enemy stun (deliberately no knockback — shoved positions read as jitter) → X/F cleaves a ~220° arc at `SWING_R` (110px) with a filled-wedge sweep visual and kill-scaled screen shake, for `SWORD_T` (~14s; kills score, trolls take 3 hits, swings bat arrows away; spawning continues while armed) → blade fades and a new stone respawns. **Waves are clear-based, not timed**: each wave is a fixed war band (`waveQuota = 8 + wave*3`, cap 30) fed through the spawn pipeline; the next wave starts only after quota, field, and warnings are all empty, via a `BREATHER` pause (~3.3s, "wave N cleared" banner). Types phase in one per wave (goblins → +wolves → +archers → +trolls) and mix thereafter; HUD shows FOES remaining. **The War-Ogre** (mini-boss, `makeEnemy('ogre')`, `drawOgre`) lumbers in once per run alongside the wave-3 band (`ogreSpawned` flag, telegraphed by a long `!` warn): a huge tusked brute that stalks, winds up (red-flash + dashed charge line), then bull-rushes a locked straight line that bounces off walls (`stalk`→`wind`→`charge`). 8 HP (`maxhp`), `kr` 30 contact-lethal, gets a `drawBossBar` (no `bossActive` — champions still allowed), no-flinch like the bosses (10-frame stagger, +20 swing/Luke reach), and on death drops a guaranteed powerup (+200, meter surge). ## Achievements
Easter-egg achievements (currently 47) defined in the `ACHIEVEMENTS` array in `app.js` (`{ id, name, desc, hint }`). Found ids persist in localStorage under `ilaird_eggs` (filtered against known ids on load). `unlockAchievement(id)` is idempotent — it saves, updates the 🥚 titlebar badge (`#ach-count` in `index.html`), and shows a toast. `toggleAchievements()` (global — wired to the badge's inline onclick; there is deliberately **no** `achievements` terminal command, badge-click only) opens a theme-aware overlay: unlocked entries show name + desc, locked show 🔒 + hint. Escape or clicking the backdrop closes it; while open, a capture-phase key handler shields other listeners. The mobile auto-launch calls `COMMANDS.gui(true)` so it doesn't unlock the `desktop` achievement. On load, `showEggNudge()` points a bobbing arrow at the badge (desktop only); it auto-hides after 60s, and is permanently dismissed (`ilaird_nudge` in localStorage) by its [x], clicking the badge, or opening the overlay — and never shows once any egg is found. When adding a new easter egg, add an `ACHIEVEMENTS` entry and call `unlockAchievement('<id>')` at the moment of discovery.

The overlay has two views rendered into the same box: `renderAchList()` (the achievement list, with a `share` button in its header) and `renderShareView()` (a 1200×630 share card rendered to canvas by `buildShareCard()` using the *current* theme colors — red in HAL mode — with copy-to-clipboard (when `ClipboardItem` exists), a download link, native share (when `navigator.canShare` supports files), and a back button). Sharing is overlay-only by design — there is no `share` terminal command. The card shows a HAL eye if `meet-hal` is unlocked.
